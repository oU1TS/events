# Walkthrough: Raid Card Linking and Highlighting

I have successfully added a deep linking system to specific raid cards, complete with a "copy link" button, scrolling, and a highlight pulse animation.

## Summary of Changes

### 1. Data Layer
- **[raids.json](Documents/GitHub/[oU1TS]/events/raids.json)**: Added a sequential `"Raid_Num"` variable (1 to 4) to the top of each raid card entry.

### 2. Frontend / Logic
- **[app.js](Documents/GitHub/[oU1TS]/events/app.js)**:
  - **Routing Updates**: Added hash detection for `#raid-<number>`. This triggers navigation to the `#past-raids` section and saves the target raid number.
  - **Copy Button Integration**: Added a button with a copy SVG icon to the top right of each card's metadata container (`.raid-meta-right`). Clicking this copies the exact absolute URL of that specific card to the clipboard (e.g. `index.html#raid-1`) and changes the icon to a checkmark for 2 seconds to provide visual feedback.
  - **Smooth Scrolling**: Implemented the `scrollToRaid` helper which centers the target raid card on the screen.
  - **Highlight Pulse**: Adds a `.highlighted` class to the target card and hooks an `animationend` listener to clean it up after the animation completes.

### 3. Styling
- **[style.css](Documents/GitHub/[oU1TS]/events/style.css)**:
  - Flex layout wrapper `.raid-meta-right` to align the status badge and copy button.
  - Copy button styles (`.copy-raid-link-btn`) with hover scale-up, press effect, active color transition, and visual success styling (`.copied`).
  - Keyframe-based pulse animation (`@keyframes highlightPulse`) that briefly increases size (by 1%), highlights the border, and adds a glow matching the current theme color (gold for dark mode, vibrant blue for light mode).

## Verification Results

1. **JSON Verification**: Parsed the `raids.json` structure using Python (`json.load`) to ensure formatting and syntax are 100% correct.
2. **SPA Routing**: Verified that the hash `#raid-` correctly maps to the corresponding `#past-raids` tab.
3. **Element Creation**: Validated that the IDs `raid-card-1`, `raid-card-2`, etc. are correctly rendered in the DOM.
