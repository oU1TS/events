# UITS Event Raiders Website Walkthrough

We have successfully created a clean, modern, and highly responsive Single Page Application (SPA) for **UITS Event Raiders** using HTML5, CSS3, and Vanilla JavaScript (ES6+).

## Changes Made

### 1. Data Layer
* Created and updated [raids.json]([oU1TS]/events/raids.json) to strictly contain the **Phoenix Summit 2026** details.
* Refactored `subEvents` from a flat string into a reusable array of collapsible objects containing a `title` and a `details` string. This structure is reusable across any event in the JSON data.
* Added exact ticket options (Exhibition Pass, Technical Pass with Swag, Student & Professional Workshops) and the complete shortened schedules for Blue Team Day (June 26) and Red Team Day (June 27).
* Utilized newline escapes (`\n`) for detailed sub-event timelines and schedules to ensure layout flexibility.

### 2. Layout Structure
* Created [index.html]([oU1TS]/events/index.html) with semantic HTML5 elements.
* Structured four tabbed views (`#home`, `#past-raids`, `#join`, `#learn-more`) matching the required hashes.
* Incorporated the UITS Event Raiders Facebook Page link (https://tinyurl.com/bdhsf7b4) on both the logo branding header and Step 1 of the "Join Us" section.
* Embedded responsive SVG icons for UI cues (social logos, checkmarks, security shields).

### 3. Styling & Theme System
* Created [style.css]([oU1TS]/events/style.css) with HSL custom properties.
* **Dark Mode (Default):** Tech-noir minimalist aesthetic utilizing black/dark gray gradients, silver-gray borders, and gold accent highlights.
* **Light Mode:** Vibrant sky blue, emerald green, and crisp white theme toggled via the body class `.light-theme`.
* Enabled smooth CSS animations (`@keyframes fadeInUp`) on SPA hash switching.
* Handled multi-line JSON breaks via CSS `white-space: pre-line` for schedules and details.

### 4. Interactive Controller Logic
* Created [app.js]([oU1TS]/events/app.js) with routing logic.
* Implemented a native URL hash router (`window.location.hash`) for bookmarking support.
* Developed asynchronous event loaders (utilizing ES6 fetch) to load past raids.
* Included strict XSS prevention mechanisms using safe DOM construction and text utilities.
* Structured responsive hamburger mobile menu event controls.
* Added interceptors for `"notes"` links to route them to our custom `render.html?file=<escaped_path>` viewer instead of opening external windows, styled with document icons.

### 5. Markdown Notes & Renderer Integration
* Created [doc/notes/phoenix-summit-2026.md]([oU1TS]/events/doc/notes/phoenix-summit-2026.md) outlining event notes, powerbank checklist, security lockdown measures, travel coordinate points, and recommended sessions.
* Modified [render.html]([oU1TS]/events/render.html) to link `./render.js` directly in the project root and removed `sw-register.js` load script.
* Updated [render.js]([oU1TS]/events/render.js) to sync themes with the main website by reading from `localStorage`.
* Created [css/render.css]([oU1TS]/events/css/render.css) with rich aesthetics, glassmorphism headers, spinner animations, and theme-variable overrides matching the primary website color palette.

## Manual Testing & Verification

1. **Routing & Navigation:**
   * Navigating to `#home`, `#past-raids`, `#join`, and `#learn-more` loads corresponding content areas smoothly.
   * Clicking the custom "View Notes" button on the Phoenix Summit card loads `render.html?file=doc/notes/phoenix-summit-2026.md` locally.
2. **Markdown Rendering:**
   * The markdown parser (`marked.js`) and compiler render headers, checkmarks, bullets, blocks, and quotes beautifully.
   * Slide-out Table of Contents drawer opens from the right side, showing nested headings. Clicking items scrolls smoothly to sections.
3. **Theme Synchronization:**
   * Clicking the theme toggle switches between Dark and Light modes on the main site.
   * Entering the Notes page automatically pulls the active theme from storage and renders it with the matching palette.
4. **Data Loading:**
   * Dynamically loads information from `raids.json` and parses multiline descriptions, links, and collapsible dropdown arrays.
