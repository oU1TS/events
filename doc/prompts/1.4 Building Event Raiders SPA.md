### Why this was happening
When the website loads, vanilla JavaScript routing initializes and redirects the URL hash to `#home`. Because `#home` is the ID of the main section, the browser naturally attempts to scroll the page to align `<section id="home">` with the very top of the screen. 

However, since our header `.header` is styled with `position: sticky; top: 0;`, it stays floating at the top of the viewport. When the browser scrolled the section to the top, the sticky header overlaid the top portion of the home section, hiding the `Initiative Active` badge directly beneath it.

### The Fix
I added `scroll-margin-top: 90px;` to the `.section` class in [style.css]([oU1TS]/events/style.css#L343-L346):
```css
.section {
    display: none;
    scroll-margin-top: 90px; /* Offset for the sticky header height */
}
```

This instructs the browser to leave a `90px` gap (matching the height of the header plus breathing room) when scrolling to any section via a hash route. The home page header, title, and badge now align at the top of the page under the sticky header.
