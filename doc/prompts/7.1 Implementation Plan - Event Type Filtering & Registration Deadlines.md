# Implementation Plan - Event Type Filtering & Registration Deadlines

We will implement a custom dropdown filter for event types beside the "Calendar View" button. Additionally, we will add registration deadline (`RegEndDate`) and event type (`Type`) fields to the events database, display them elegantly on the cards, and ensure high-fidelity responsive behavior.

## Verification Plan

### Automated/Local Build Checks
- Run standard browser checks to ensure there are no syntax or console errors.

### Manual Verification
- **Aesthetics & Theme Sync:** Inspect custom dropdown design in both dark and light modes.
- **Responsiveness Check:** Test mobile responsive viewport. Shrink the browser width down to 380px and ensure the two buttons wrap elegantly to two rows and take full width *only* at and below 380px. Check that they remain in a single row above 380px.
- **Filtering Capability:** Click on different event types in the dropdown and confirm that the list updates dynamically, only displaying the selected category.
- **Reg Deadline Display:** Verify that closed registration deadlines (like Phoenix Summit 2026) display in muted colors with `(Closed)`, while open ones display in vibrant green.
- **Deep Linking Compatibility:** Navigate to a deep link (e.g. `#raid-2`) and verify it resets the filter, shows the card, scrolls to it, and runs the highlight pulse animation.
