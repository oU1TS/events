# Interactive Campaign Calendar View

This plan outlines the design and implementation details for adding a minimalistic, premium calendar view of events inside the "Raid Campaigns" section of the website. It allows users to visualize upcoming and past raids in a square calendar view, highlight event dates, handle date overlaps, and view details or navigate directly to the target cards.

## User Review Required

> [!NOTE]
> The calendar sizing will be strictly constrained to `50vh` height/width on mobile and `60vh` height/width on desktop, keeping it perfectly square-shaped as requested.

## Proposed Changes

### Data Layer (`raids.json`)
We will add a `"startDate"` property (formatted as `YYYY-MM-DD`) to each raid object to enable easy calendar date matching.

#### [MODIFY] [raids.json](Documents/GitHub/[oU1TS]/events/raids.json)
Add `"startDate"` to each event:
- **Raid 7**: `"startDate": "2026-07-10"`
- **Raid 6**: `"startDate": "2026-07-31"`
- **Raid 5**: `"startDate": "2026-07-20"`
- **Raid 4**: `"startDate": "2026-06-30"`
- **Raid 3**: `"startDate": "2026-07-10"` (overlapping with Raid 7)
- **Raid 2**: `"startDate": "2026-07-06"`
- **Raid 1**: `"startDate": "2026-06-23"`

---

### Layout Layer (`index.html`)
We will add a "Calendar View" button to the `#raids` section header, and a modal backdrop containing the square calendar grid and day-events list overlay.

#### [MODIFY] [index.html](Documents/GitHub/[oU1TS]/events/index.html)
- Place an "Open Calendar" button in the `#raids` section header.
- Add a modal container (`#calendar-modal`) at the bottom of the section.
- Inside the modal, add:
  - Navigation controls (Previous/Next Month buttons, Month-Year label).
  - Day names header (Sun - Sat).
  - Days grid container.
  - A slide-up/fade-in vertical pop-up overlay (`#day-events-overlay`) to show a list of events when clicking a highlighted cell.

---

### Styles Layer (`style.css`)
We will style the calendar components to fit the dark-mode aesthetic with smooth transition animations.

#### [MODIFY] [style.css](Documents/GitHub/[oU1TS]/events/style.css)
- Style the "Calendar View" button using a tactile layout.
- Implement the modal backdrop with glassmorphic blur and opacity animations.
- Design the calendar container to be perfectly square (using `aspect-ratio: 1/1` and height `50vh` for mobile, `60vh` for desktop).
- Setup the calendar days as a CSS grid of 7 columns.
- Create styling for:
  - Highlighted days (`background: var(--accent-gradient)`, smooth hover scaling).
  - Today's date (distinctive border style).
  - Overlap event count badge (top-right absolute position badge).
  - Day-events translucent overlay list with card-style items that react to hovers.

---

### Logic Layer (`app.js`)
We will handle calendar generation, event matching, overlaps, and user navigation interactions.

#### [MODIFY] [app.js](Documents/GitHub/[oU1TS]/events/app.js)
- Extend routing to close the calendar modal if the user navigates or redirects.
- Add a calendar controller module:
  - Cache the calendar elements.
  - Listen for "Calendar View" open/close events.
  - Dynamically generate the grid cells for the currently selected month and year.
  - Parse `"startDate"` for each event using timezone-safe local date parsing.
  - Match events against the generated days. If events match:
    - Highlight the day cell.
    - If overlaps exist (>= 2 events), render a small badge with the event count.
    - Attach a click handler that opens the day-events overlay.
  - Pop-up overlay logic:
    - Clear and render the matching events as cards inside the list.
    - Clicking on an event card sets the hash `window.location.hash = '#raid-' + raid.Raid_Num`, which automatically closes the calendar/popups and scrolls the target raid card into view.
