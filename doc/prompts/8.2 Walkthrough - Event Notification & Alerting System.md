# Walkthrough - Event Notification & Alerting System

We have implemented a client-driven notification and cache-management architecture that integrates local tracking, a premium on-screen toast warning system, OneSignal Web Push SDK, and automated GitHub Actions workflows.

---

## 🛠️ Changes Implemented

### 1. Registry & Caching System
- **[NEW] [tracker.json](Documents/GitHub/%5BoU1TS%5D/events/tracker.json)**: Created a tracking registry managing synchronization states. It monitors the total active event count, the last notification dispatch timestamp, and a list of upcoming active registration deadline reminders (`activeReminders`).
- **[MODIFY] [app.js](Documents/GitHub/%5BoU1TS%5D/events/app.js)**: 
  - Added an aggressive caching logic layer. On document load, it bypasses standard browser caching to fetch a fresh `tracker.json` (`cache: 'no-cache'`).
  - Compares the remote metadata against `localStorage` values under key `ev_tracker`.
  - If a newer configuration is available (remote `eventsCount` is greater), it flushes local caches and fetches a clean copy of `raids.json` using `{ cache: 'no-cache' }`, updating the local tracker state.
  - Added logic to evaluate deadlines in `activeReminders` and trigger localized systems + UI warnings exactly 1 day prior to any registration end date. It sets a local flag to prevent duplicate alerts.
- **[MODIFY] [style.css](Documents/GitHub/%5BoU1TS%5D/events/style.css)**: Appended styles for the `.toast-container`, `.premium-toast`, glowing accent lines, custom buttons, progress bars, and animations (such as slide-in and timeout decay).

### 2. OneSignal Integration
- **[MODIFY] [index.html](Documents/GitHub/%5BoU1TS%5D/events/index.html)**: Integrated OneSignal Web Push SDK (`OneSignalSDK.page.js`) inside `<head>`. Set initialization block styled to match the dark theme and gold accent properties of the platform.
- **[NEW] [OneSignalSDKWorker.js](Documents/GitHub/%5BoU1TS%5D/events/OneSignalSDKWorker.js)**: Created service worker at the root directory level importing the OneSignal service worker script to handle background registration.

### 3. CI/CD & Automated Push Dispatch
- **[NEW] [send-push.js](Documents/GitHub/%5BoU1TS%5D/events/.github/scripts/send-push.js)**: Written a Node.js script supporting:
  - **New Raid Mode**: Triggers on push. Finds the most recent raid entry, dispatches a push notification with a tailored title/description, and updates `tracker.json`.
  - **Deadline Scan Mode**: Runs daily on a cron trigger. Scans the reminders, determines if any are closing tomorrow, dispatches warnings, and marks them as sent in `tracker.json`.
- **[NEW] [notify.yml](Documents/GitHub/%5BoU1TS%5D/events/.github/workflows/notify.yml)**: Configured the GitHub Actions pipeline to run Node.js, run `send-push.js` (automatically switching between modes based on trigger), and securely pass secrets before committing back `tracker.json` changes.
