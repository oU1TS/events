# UITS Event Raiders Website Implementation Plan

Create a premium, modern, and highly responsive Single Page Application (SPA) for "UITS Event Raiders" using HTML5, CSS3 (with rich variable-based Dark/Light themes), and Vanilla JavaScript (ES6+), fetching data from a separate JSON file (`raids.json`).

## User Review Required

> [!IMPORTANT]
> **Theme Palette Specifications:**
> * **Dark Mode:** Minimalist black & white palette. We will use a sleek tech-inspired design (pitch black/deep charcoal backgrounds, bright white text, silver-grey glassmorphism, and minimal glowing outlines).
> * **Light Mode:** Vibrant blue, green, and white theme. We will implement high-contrast components using fresh teal/emerald accents (`--success-color`) and sapphire blue (`--accent-color`).
>
> **SPA Router:**
> We will implement a URL hash-based router (`#home`, `#past-raids`, `#join`, `#learn-more`) so that bookmarking works natively and the browser back/forward buttons function as expected.

## Proposed Changes

We will create four core files in the project root:

---

### [NEW] [index.html]([oU1TS]/events/index.html)
The page skeleton featuring:
* A modern head with Google Fonts ("Outfit" for headings, "Inter" for body) and FontAwesome (or SVGs) for clean UI icons.
* A sticky minimalist header containing:
  - The branding logo/title: **UITS Event Raiders** (linking to their facebook page).
  - Navigation links: Home, Past Raids, Join Us, Learn More.
  - A custom-designed, tactile, and animated Theme Toggle Switch (Dark vs Light).
* Section containers matching the router hashes:
  - `<section id="home">`: Hero area with a dynamic gradient text background, subtitle, and CTA buttons.
  - `<section id="past-raids">`: Grid/Timeline skeleton to render past events loaded from `raids.json`. Includes a skeleton loading state.
  - `<section id="join">`: Interactive onboarding steps (Step 1: social profiles, Step 2: Student ID verification preview/info, Step 3: mutual contact requirements) and a privacy disclaimer.
  - `<section id="learn-more">`: Grid of external resources featuring the primary highlight card pointing to the oU1TS Community Project.
* Modern footer.

---

### [NEW] [style.css]([oU1TS]/events/style.css)
The design system using custom CSS properties:
* Define CSS variables under `:root` (Dark Theme as default) and `.light-theme` class.
* Modern layout styles utilizing Flexbox and CSS Grid.
* Glassmorphism effects (`backdrop-filter`) for cards and headers.
* Custom-designed theme toggle layout.
* Custom scrollbars, focus rings, hover lift effects, and CSS transitions on property changes for a premium, tactile feel.
* Use `white-space: pre-line` for detailed descriptions and sub-events to respect newline characters (`\n`) parsed from the JSON.

---

### [NEW] [app.js]([oU1TS]/events/app.js)
The client-side controller containing:
* SPA Hash Routing:
  - Listens to `hashchange` and `DOMContentLoaded` to hide/show sections.
  - Smooth active-tab states on header links.
* Dynamic Data Fetching:
  - Asynchronously loads `raids.json` using `fetch()`.
  - Dynamically builds HTML elements for each raid card.
  - Graceful error handling (displays custom warning banner in case of fetch failure) and skeleton loading states.
* Theme Management:
  - Toggles the `.light-theme` class on the `<body>` element.
  - Remembers the user's theme preference in `localStorage`.
  - Honors system preferences (`prefers-color-scheme`) as a fallback.

---

### [NEW] [raids.json]([oU1TS]/events/raids.json)
Sample JSON containing mock history of past event raids:
* Must contain field details like `title`, `dateRange`, `subEvents` (with newline `\n` linebreaks), `fee`, `venue`, `links` (containing name-url key-value maps), and `details` (with `\n` linebreaks).

## Verification Plan

### Automated Tests
* We can validate file syntax and formatting using browser tools or simple static testing.

### Manual Verification
* Run a local HTTP server to host the project.
* Open the browser and test:
  1. Routing: Click all navigation links, home CTAs, and test using back/forward buttons.
  2. Responsive Design: Verify viewports from mobile (320px) to ultra-wide desktop.
  3. JSON Loading: Ensure data parses and renders perfectly. Verify newline line breaks work.
  4. Theme Toggle: Test transition animation and persistence of the theme across page refreshes.
