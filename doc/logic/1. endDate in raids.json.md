### Planner Response

The `endDate` logic in [app.js](file:///c:/Users/gsmur/Documents/GitHub/[oU1TS]/events/app.js) is responsible for dynamically updating and rendering the status of each event on the page.

Here is the block of code from the rendering function:

```javascript
// Dynamically check and update Status from Future to Past if the date has passed
if (raid.endDate) {
    const today = new Date();
    today.setHours(0, 0, 0, 0);
    const checkDate = new Date(raid.endDate);
    checkDate.setHours(0, 0, 0, 0);
    if (today > checkDate) {
        raid.Status = "Past";
    }
}
```

### Breakdown of how it works:

1. **`if (raid.endDate)`**:
   First, it checks if the event object loaded from `raids.json` contains the optional `"endDate"` key (e.g., `"endDate": "2026-06-27"`).

2. **Normalizing Today's Date**:
   ```javascript
   const today = new Date();
   today.setHours(0, 0, 0, 0);
   ```
   This gets the current local date and time and resets the time parts (hours, minutes, seconds, milliseconds) to midnight (`00:00:00.000`). This normalization prevents time-of-day offsets from throwing off the date comparison.

3. **Normalizing the Target End Date**:
   ```javascript
   const checkDate = new Date(raid.endDate);
   checkDate.setHours(0, 0, 0, 0);
   ```
   This parses the `"endDate"` string into a JavaScript `Date` object and normalizes its time component to midnight as well.

4. **Status Promotion**:
   ```javascript
   if (today > checkDate) {
       raid.Status = "Past";
   }
   ```
   If today's date has strictly passed the event's end date, the script overrides/updates the event's status to `"Past"`. For example, if an event was statically marked `"Status": "Future"` in the JSON, it will automatically shift to `"Past"` once that day is over.

5. **UI Rendering**:
   The value of `raid.Status` is subsequently used to append the appropriate badge chip:
   * `"Future"` renders a green/success badge (`.status-future`).
   * `"Past"` renders a muted gray badge (`.status-past`).
